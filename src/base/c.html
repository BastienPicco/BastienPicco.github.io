Introduction

La syntaxe du C est plus complexe que celle de Python. Toutefois, cela permet, en partie, d'avoir plus de contrôle et de mieux comprendre chaque action. De plus, un certain nombre d'autres langages sont syntaxiquement proches du C.



Tabulation et Point-Virgule

En C les tabulations, espaces et retour à la ligne n'ont aucune valeur. Vous pouriez écrire l'intégralité de votre code sur une suel ligne. Toutefois, nous ne le recommadons pas, car il serait, alors, très difficile à comprendre et débugger. De même, il est fortement conseillé de garder l'habitude d'indenter votre code.

A la place les groupes de code sont regroupé entre {accolade}. Et chaque ligne est fini par un point virgule.



Déclaration de Type

En C, contrairement à en python, il faut déclarer le type de chaque variable. Ainsi, la déclaration et l'assignation d'une variable peuvent se faire simultanément ou non, suivant le besoin. 


                    float monFloatant;
                    int monEntier = 5;
                    float unAutreFloatant = 1.2;


Boucles et Conditions

Sans plus attendre, voici la structure des différentes notions vues en Python. 


                    //Ceci est un commentaire
                    for(int i = 0; i < 10; i++){
                        maFct(); // code quelconque
                    }

                    int a = 0;
                    while(a < 10){
                        maFct();
                        a++;  // équivalent à : a = a + 1; ou a += 1;
                    }

                    if( condition1 ){
                        maFct1();
                    }
                    else if( condition2 ){ // Pas de contraction en elif
                        maFct2();
                    }
                    else{
                        maFct3();
                    }

Pour la boucle for, la structure est la suivante :
for(déclaration et initialisation de la variable compteur; condition d'arrêt; action à réaliser à la fin de chaque boucle, généralement incrémenter la variable de 1){code}



Pointeur

Les pointeurs permettent de différencier entre un agissement sur une copie de la variable ou sur la variable elle-même. Ils permettent de faire référence non pas à la valeur de la variable mais à l'adresse mémoire de la variable.
On utilise & pour accéder à l'adresse d'une variable et * pour le type : adresse mémoire. 

                int a = 0;
                int * p; // p est un pointeur vers un adresse contenant un entier
                p = &a; // p prend la valeur de l'adresse de a

En C, les listes fonctionnent explicitement avec des pointeurs. C'est à dire qu'une liste va "être égale" à l'adresse de son premier élément, qui lui-même connaît l'adresse du suivant. Attention : les listes ont des tailles prédéfinies et il ne faut pas les dépasser (ni pouvoir le faire). De plus, dans le cas de char le dernier emplacement doit être réservé au caractères fin de chaîne (ou caractère vide \0).

Compilation

A présent que vous avez une idée de comment écrire en C, il faut encore l'écrire et le compiler. Pour l'écriture, rien de bien compliqué, n'importe quel éditeur de texte suffit. Toutefois, il est préférable d'utiliser des éditeurs adaptés au code. Je conseille, personnellement, VSCodium.
Le code C, à la différence du python qui est interprété, doit être compilé puis exécuté. Pour se faire, le plus simple est d'utiliser le terminal unix.
Positionnez-vous dans le répertoire contenant le code.c et tapez :
gcc -Wall -o nomDeLExecutable code.c
Cela permet, dans l'ordre, de compiler un code c (gcc), en affichant tous les warning (-Wall) et en nommant l'exécutable formé (-o) en nomDeLExecutable, contenu dans code.c.
Sans le -o le nom par défaut est a.out.
Maintenant que votre code est compilé, et s'il n'y a pas d'erreur à débugger avant, vous pouvez l'exécuter :
./nomDeLExecutable

Il faut recompiler le code à chaque fois qu'il subit des changements que vous voulez tester.
