Introduction

Préambule

Nous allons, ici, apprendre à utiliser les commandes de base en Unix.


Préparation

Unix est un OS (Operating System), au même titre que Windows et macOS.
Son principal avantage en programmation est sa programmabilité (en tant qu'Open Source) ainsi que sa console.
Note :
La plupart des commandes unix ont un équivalent proche pour mac. Sur Windows il est possible d'accéder à une configuration Unix à l'aide d'une VM (Machine virtuelle) ou d'une WSL (Sous-système Windows pour Linux). La VM est comme une application que l'on ouvre pour avoir un portail vers unix. Alors que la WSL permet d'avoir unix directement d'installer à côté de windows.
Si vous utilisez un Windows, vous allez devoir choisir une des 2 méthodes pour continuer. La VM permet, sans doute, plus de contrôle au niveau de l'environnement et la WSL est plus pratique car on peut accéder à tous ces dossiers personnels, alors qu'avec une VM on ne peut accéder, facilement, qu'à ce qui se trouve de l'autre côté du portail.
A titre personnel j'utilise une WSL, car cela me paraissait plus simple, et je n'ai pas encore trouvé de cas d'utilisation limitant.

Bien, maintenant que vous avez accès à un terminal (autre que celui de Windows), nous pouvons commencer.



man et arguments

Le man sert à connaître l'utilité d'une commande ainsi que ses arguments.

De manière générale, pour toute commande, vous pouvez taper : man nomDeLaCommande afin d'avoir des informations sur cette commande. Le man (pour manuelle) est très long. Vous pouvez vous déplacer vers le bas avec la touche espace et sortir avec la touche q. Vous pouvez aussi utiliser le man en ligne.

Un argument est sous la forme -uneLettre (ex : -s, -a, -l, ...). Il permet de préciser la commande.



ls & cd

ls affiche tous les dossiers et fichiers du répertoire courant.

cd prend en argument direct (sans tiret) un chemin absolue ou relatif et permet de se déplacer à cet endroit.



Pipeline & Redirection

Un pipeline a|b permet d'envoyer la sortie de la commande a à b

une redirection a>b permet de renvoyer la sortie de a (print) vers le document b, qui sera créé s'il n'existe pas.



sudo

sudo est le "mot" devant se trouver devant toutes commandes nécessitant des droits d'administration.
L'exemple le plus utile est : sudo apt install nomModule afin d'installer un module.



Extension

En unix, il n'y a pas réellement d'extension. Il faut tout de même continuer à les marquer afin de reconnaître les fichiers, mais c'est tout.
Autrement dit, vous pouvez lancer monCode.py avec un compilateur c. Cela soulèvera des erreurs mais ne sera pas impossible. C'est pourquoi il faut penser à les indiquer tout de même, afin de savoir comment les traiter.

Note : Pour exécuter un code python il faut avoir le module python3 et taper :
python3 monCode.py, en étant dans le répertoire où se trouve monCode.py.



Fin
