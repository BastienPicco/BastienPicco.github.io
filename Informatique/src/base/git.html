Introduction

Github est un site permettant de sauver et tracer ses codes open-source. Ceci permet aussi de travailler à plusieurs sur un même code. Pour cela, chacun va avoir sur sa machine une copie local du code qu'il pourra modifier et compiler. Une fois satisfait avec sa partie, il envoie le code sur le site qui se charge, presque entièrement tout seul, de traquer ce qui a changé et de fusionner correctement les différentes parties.

Afin d'éviter au maximum les fusions difficiles, je recommande de se diviser le travaille en fichiers. Toutefois, la plupart des problèmes peuvent assez facilement être réglés avec une bonne interface graphique.



Commandes de Bases

Je vous conseille de créer un dépôt git (projet) pour tester comment tout cela marche. Pour cela vous aurez besoin de plusieurs fichiers. Toutefois, git fonctionne avec tout, pas seulement le code. Aussi les textes et images.

Pour commencer un nouveau projet, commencez par créer un nouveau dépôt git.
Une fois créer, copier son url, aussi accessible à l'aide du bouton code en vert.
Ouvrez le terminal unix et positionnez-vous là où vous voulez créer votre copie local du projet. et tapez :
git clone votreURL afin d'initialiser le répertoire sur votre machine.
Vous pouvez à présent effectuer toutes les modifications et ajouts que vous voulez.

Une fois satisfait par vos changements, pour le moment, il faut regrouper les changements que vous voulez envoyer sur le site et nommer ce groupe afin d'identifier ces changements.
git add nomFichier1 nomImage2 nomCode.c pour ajouter ces fichiers, ou
git add * pour ajouter tous les changements, puis :
git commit -m "nom / Description de vos changements"

Il ne reste plus qu'à envoyer ces changements sur le site, pour cela :
git push, cette action vous demandera votre identifiant et token.

Enfin, pour récupérer des changements, fait par une autre personne, ou si vous travaillez sur différentes machines :
git pull, attention : cela ne marche que si le projet a déjà été cloné.



Merge

Si 2 personnes, ou plus, modifient le même fichier, git sera, le plus souvent, capable de fusionner les changements correctement. Le cas le plus problématiques est lorsque les 2 personnes modifient les même lignes. Dans ce cas, il faudra indiquer à git lequel des changements garder.

En ligne de commande cela peut vite devenir difficile. C'est pourquoi je recommande l'éditeur VSCodium qui permet de gérer les fusions directement : les 2 propositions apparaîtront dans l'éditeur et on peut modifier et garder celle que l'on veut.
